# CARLA环境修复说明

## 修复的问题

### 1. ✅ NPC车辆不自动运行的问题

**问题原因：**
- Traffic Manager可能没有正确初始化
- 车辆生成后没有立即设置自动驾驶
- 同步模式下需要等待几帧才能开始移动

**修复方案：**
```python
# 1. 确保Traffic Manager正确初始化
traffic_manager = client.get_trafficmanager()
traffic_manager.set_global_distance_to_leading_vehicle(2.5)
traffic_manager.set_random_device_seed(0)

# 2. 获取正确的端口
tm_port = traffic_manager.get_port()

# 3. 为每辆车设置自动驾驶
for vehicle in npc_vehicles:
    vehicle.set_autopilot(True, tm_port)

# 4. 等待车辆开始移动（同步模式需要tick，异步模式需要sleep）
if args.sync:
    for _ in range(10):
        world.tick()
else:
    time.sleep(1.0)
```

## 使用方法

### 运行完整测试

```bash
python autonomous_driving_test.py --num-vehicles 30 --num-walkers 20 --sync
```

**参数说明：**
- `--num-vehicles`: NPC车辆数量（默认30）
- `--num-walkers`: NPC行人数量（默认20）
- `--sync`: 使用同步模式（推荐用于数据采集）
- `--autopilot`: Ego Vehicle使用自动驾驶
- `--map`: 指定地图（如Town01, Town10等）

### 验证NPC车辆运行

运行脚本后，观察CARLA窗口：
1. NPC车辆应该在几秒内开始移动
2. 车辆会遵循交通规则
3. 车辆会避让其他车辆

如果车辆仍然静止：
1. 检查Traffic Manager是否正确初始化
2. 等待更长时间（可能需要5-10秒）
3. 检查是否有碰撞导致车辆卡住

### 验证数据采集

运行脚本后，检查数据目录：
```bash
dir carla_data\rgb
dir carla_data\depth
dir carla_data\semantic
dir carla_data\lidar
dir carla_data\radar
```

应该看到文件在不断生成。

## 常见问题

### Q: NPC车辆仍然不移动？
A: 
1. 确保使用 `--sync` 参数
2. 等待10-15秒让车辆开始移动
3. 检查Traffic Manager是否正确初始化
4. 尝试重新生成车辆

### Q: 数据没有保存？
A:
1. 检查磁盘空间
2. 检查目录权限
3. 查看错误日志
4. 确保传感器正确安装

### Q: 如何查看采集了多少数据？
A: 脚本结束时会自动打印统计信息，包括：
- 每个传感器采集的帧数
- 错误次数
- 最后帧号
